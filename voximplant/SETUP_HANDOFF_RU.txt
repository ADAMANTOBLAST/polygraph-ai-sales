Настройка переадресации на специалиста (+375291252472)
=====================================================

1) Voximplant — сценарий
------------------------
- Откройте приложение (например flex-n-roll...adamantoblast.voximplant.com).
- Scenarios → создайте или отредактируйте сценарий для входящих на тестовый номер.
- Вставьте код из файла scenario_elevenlabs_handoff_by.js (скопируйте целиком в редактор JS).
- В начале сценария обязательно:
  - XI_API_KEY — ключ ElevenLabs (xi-api-key).
  - ELEVENLABS_AGENT_ID / ELEVENLABS_BRANCH_ID — как в вашей консоли ElevenLabs.
  - SPECIALIST_PSTN = "+375291252472" (уже в файле, можно не менять).
  - OUTBOUND_CALLER_ID — номер, с которого Voximplant имеет право звонить наружу (часто ваш входящий DID, например +7499...). Должен быть верифицирован в аккаунте.
- Сохраните и опубликуйте сценарий.

2) Voximplant — маршрут
-----------------------
- Routing → правило для входящего номера (+74999384362 или тестового) → укажите этот сценарий.
- Убедитесь, что включён модуль ElevenLabs (Voice AI) для приложения.

3) ElevenLabs — агент
---------------------
- Agents → ваш агент → Knowledge base: блок «【Компания】» … «【Роли и передача трубки】» из elevenlabs_voice_agent_prompt_handoff.txt (или весь файл как документ).
- System prompt: файл elevenlabs_voice_system_prompt.txt целиком (или дубликат блока в elevenlabs_voice_agent_prompt_handoff.txt).
- Tools → добавьте именно Client tool (не Server tool и не Webhook):
  - Имя: transfer_to_specialist (точно так же, как в JS сценарии).
  - Описание: блок «Client tool» в elevenlabs_voice_agent_prompt_handoff.txt (обновлённое: сначала вызов, без «передачи» без вызова).
- Сохраните и опубликуйте смену (в т.ч. branch, если используете).

4) Проверка
-----------
- Позвоните на входящий номер.
- Скажите фразу про макет/ТЗ/технолога — в логах сценария должна появиться строка «ClientToolCall payload: …», затем исходящий на +375291252472.
- Агент говорит «передам», но в логах нет ClientToolCall — модель не вызывает tool: проверьте тип инструмента (только Client tool), имя transfer_to_specialist и обновите System prompt + описание tool из elevenlabs_voice_system_prompt.txt / elevenlabs_voice_agent_prompt_handoff.txt.
- ClientToolCall есть, но нет исходящего — смотрите SPECIALIST_PSTN, OUTBOUND_CALLER_ID и строку SPECIALIST_PSTN Failed в логах (верификация Caller ID, баланс, маршрутизация на +375).

5) Безопасность
---------------
- Не коммитьте реальный XI_API_KEY в публичный репозиторий; в репозитории оставляйте пустую строку и вставляйте ключ только в панели Voximplant.
